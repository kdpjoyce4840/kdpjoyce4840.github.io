Learning, Practice, and Growth

These reflections highlight what I’m learning as I develop skills in instructional design and educational technology—what worked, what I’d change, and how I’m applying research-informed practices to create accessible, engaging learning experiences.

Designing for Accessibility First

I’ve adopted an accessibility-first mindset. For each page and activity, I now run through a quick checklist:

  • Descriptive page titles and meaningful headings (H1 → H2 → H3)
  • Color contrast that exceeds WCAG AA (Tiffany/Charcoal/Navy palette)
  • Alt text for images and icons only when they convey meaning
  • Keyboard navigability and visible focus states
  • Plain-language microcopy and consistent link styles

This has improved clarity and reduced cognitive load for all learners—not just those who rely on assistive technology.

Structuring Content for Sense-Making

I’m experimenting with chunking, advance organizers, and progressive disclosure to support comprehension. A consistent layout with short intros, clear objectives, and small, “do-able” tasks helps learners know exactly where they are and what’s next.

I’ve also tightened typography and spacing to make scanning easier, using section dividers (thin navy rules) and consistent margins to guide the eye.

Iterating with Feedback

Peer and instructor feedback has shaped my process. I now prototype quickly, test with a few users, and iterate:

  1. Sketch → low-fidelity prototype for structure
  2. Style → apply palette, spacing, and components
  3. Refine → improve language, add interaction, verify accessibility

This cycle keeps me focused on learner needs while improving quality with each revision.

Tools & Workflow

Beyond HTML/CSS and GitHub Pages, I’m building a toolkit that supports rapid design and iteration:

  • Versioning & hosting: Git + GitHub Pages
  • Wireframes & mockups: Figma / pen-and-paper
  • Checks: color contrast & keyboard testing
  • Documentation: simple style guides and page patterns
