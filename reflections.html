Reflections

Learning, practice, and growth across design and technology.

Designing for Accessibility First

I’ve adopted an accessibility-first mindset. On every page and activity, I run a quick checklist so that clarity and access aren’t “nice to haves” but the default.

My checklist includes descriptive page titles, a logical heading order (H1 → H2 → H3), strong color contrast, meaningful alt text, keyboard navigability with visible focus states, and clear, consistent link styles.

The impact has been visible: fewer questions about “where to click,” easier scanning for all users, and content that holds up better on small screens or assistive tech.

Structuring Content for Sense-Making

I’ve leaned into chunking and progressive disclosure—short intros, explicit objectives, and “do-able” steps that reduce cognitive load.

Using consistent spacing, gentle dividers, and predictable patterns makes the interface feel calmer. Learners don’t spend energy figuring out the layout; they spend it on the task.

Iterating with Feedback

Feedback loops are now part of my normal workflow. I prototype quickly, test with a few people, and iterate in tight cycles.

My rhythm looks like this: sketch for structure → apply palette and spacing for style → refine language and interactions → verify accessibility. Each pass gets me closer to something that’s clear and usable.

Tools & Workflow

Beyond HTML/CSS and GitHub Pages, I’m building a small toolkit for reliable delivery: version control with Git, quick wireframes in Figma (or pencil!), contrast checks, keyboard testing, and lightweight style guides.

The goal is speed with quality—get something in front of users fast, then keep polishing.
